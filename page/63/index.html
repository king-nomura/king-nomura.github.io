マザーを入れ替えるとWindowsが大変なことに

マザーボードを入れ替えておきながらOSの再インストールはしたくないという、甚だ自己中心的な行動に対してかのMicro$oftが真摯な態度をとる訳が無く。

まずOSの上書きインストールだが、インストールが「残り34分」でストップ。デバイスの認識で引っかかっているっぽい。なんとインストール途中でShift+F10キーを押すとコマンドプロンプトが立ち上げ、ここから「compmgmt.msc /s」を実行するとデバイスマネージャが使えるようになるそうだ。すごいなぁ、この技。で、どうやらBHAの仮想ドライブユーティリティが邪魔をしていたようだ。そりゃそうだ、そんなドライブは物理的には存在しないのだから。その他にも古いマザーボードに関連したデバイスがあったので「無効」にしてからインストールやり直し。無事、OSの再インストール完了。

次にOSの認証……、ってネットワークに繋がっていないのにどうやってやるんだよ、バカじゃねーの？というわけでドライバ類のインストールやネットワーク周りの設定をやるわけだが、このマザーボードにオンボードなNICがうまく動いてくれない。固定IP振って、ゲートウェイにPINGするも返答なし。ループバック(127.0.0.1)も自分自身(割り振った固定IP)もPINGが帰ってくるのに……。

いや、ハマッタハマッタ。ドライバを入れ直したりしてみたが繋がったり繋がらなかったりする。動くときはサクサク動く、で再起動するとダメだったり。

で、ネットで調べたところどうもRealtek RTL8169/8110 Family Gigabit Ethernet NICのLAN速度の判定(Link SpeedをAutoにする)は良くないということが分かった。で、試しにLink Speedを10Mbps/Full Duplexにしてみたら、あらま、サクっとゲートウェイまでPINGが通った。なんですかこれは、酷いなぁ。

気を取り直してGIGABYTEのドライバCDに入っていたユーティリティの類を試しに入れてみる。が、ATIのユーティリティの1つにトロイの木馬が仕込まれている疑いがある、とウチのアンチウィルスソフト(Avast)が警告してきた。うーむ、GIGABYTEさん、これマジですか？誤判定であったとしても情報提供してくれないと問題アリですぜ。

まぁ、なんだかんだで、マザーボード入れ替え前と同じWindowsが使えるようになったのは便利。アプリケーションの入れ替えが要らなかったので見た目が全く一緒、というのは切なくもあり。CPUが速くなったり、メモリがほぼ倍増したりしているはずなのだが、……とりあえず実感なし……。

CPU、マザーボード取替え

先日メインPC(鷲羽Blackと呼んでいた)のCPUがお亡くなりになった件。新しく買ってきたマザーの方のFSBの設定が間違っていたのが原因のようだ。なんというか……、初歩的な。

さあ、気持ちを入れ替えて。買ってきました新CPUと新マザー。

CPU : AMD Athlon64x2 4400+(TDP65W)

M/B : Gigabyte GA-MA69GM-S2H(AMD690G)

Memory : DDR2-800 1GB x 2(CencuryMicro)

さすがに64bitOSは未だ手にしていないが、アーキテクトががっつり変わっちゃうわけで当然ながらそれまで入っていたWindows XPが立ち上がらない。修復インストールを試みるが何やら認識できないボリュームがあるとか何とか……、はて？

すっかりはまってしまって、MBRの書き直しやら色々試したのだが、結局IDEに刺さっていたHDD(2台めとして利用)が悪さをしていただけであった。修復インストールの場合、Windows的に不要なドライブは外しておいたほうが良さそう。

もう一つ困ったことがあって、修復インストールなどの作業をする際にはBluetooth経由でキーボード(初代diNovo)が認識してくれない。BIOS画面では使えるんだが、一体どういう仕組みなんだろう。仕方がないので古い日本語キーボードを英語キーボードだと嘘をついて作業を行っている。

インストール作業は続行中。うまく行くといいな。

買い換えねばならない事情

そろそろCPUを買い換えたいなぁ、と考えていた。現在のCPUはAthlon XP 2800+。CPUを買い換えるとなると、マザーボードもメモリも買い換えねばならないのでちょっとした出費だ。だがこのCPU、まだまだ現役引退というほど低スペックでもない。そこでCPUやメモリを人に譲ろうと思いついた。ただし譲る先のマシンがMicroATXなのでこのままでは譲れない、MicroATXのマザーボードを入手する必要がある。

今更SocketAのマザーはなかなか手に入らない。生産している唯一のマザーのスペックやメーカーがどうにもピンと来なかったので中古を探して、この週末にようやく納得のいくMicroATXマザーを入手した。今使っているマザーと比べてスペック的に見劣りするところも少ないので、初期不良を調べるついででマザーを取り替えてしまうことにした。

で、早速乗せ換えてみたのだが、これがウンともスンともいわない。ハテ？と思い一度元のマザーにCPUを戻してみると、なんと起動しない。

やってしまった。CPU、死亡。

わずか数分の間に、この差し替えのどこかでCPUを壊してしまったようだ。

さぁ困った。今更こんなしょぼくれたCPUを中古で探してくるか。それとも思い切って新しいCPUとマザーボードとメモリを買ってくるか。譲ろうと思っていた先のPCはどうしようか。せっかく買ってきたマザーボードはどうしようか。
